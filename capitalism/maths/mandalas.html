"How long does rendering a thing such as this take?", one might ask. Two answers, the short one - too sodding long, the not-quite-as-short one is that due to the process' inherent randomness one need run the algorithm body many times, an acceptable amount I found was:
<AsciiMath>log(n_max) = |~log(prod_i size_i)~| + 1</AsciiMath>,
(which still doesn't perfectly fill more complex corner setups), so for <AsciiMath>size = [900, 900, 900] => n_max = 10^10</AsciiMath>, which takes about <AsciiMath>7 1/2 h</AsciiMath> on my PC if I try to not cook it (and I generally do).
